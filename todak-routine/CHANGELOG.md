## 문제 해결 기록

### [PROB-007] 앱 아이콘에 흰 여백이 다시 생김 (2026-07-21)

**문제 (Problem)**: 홈 화면 앱 아이콘 바깥에 흰 여백이 보였다.

**원인 (Cause)**: 매니페스트는 아이콘을 `purpose: "any maskable"`로 선언(full-bleed 필요)하는데, 이후 아이콘 그림이 모서리가 흰색인 둥근 카드 버전으로 교체되면서 maskable 마스킹 시 그 흰 모서리가 여백으로 드러났다.

**해결 방법 (Solution)**: 아이콘 네 모서리의 흰색 영역을 카드 크림색(#fff1dd 계열)으로 코너 flood-fill 처리해 full-bleed로 만들었다. 내부 흰 구름·반짝임은 카드 테두리로 분리돼 있어 그대로 유지했고, `app-icon`/`icon`의 192·512px에 모두 적용했다.

### [PROB-006] 빠른 기록으로 낮잠 종료 시 다음 낮잠 시간이 실제 종료를 반영하지 않음 (2026-07-16)

**문제 (Problem)**: 낮잠을 빠른 기록 버튼으로 시작/종료하면 실제 종료 시각을 입력했는데도 다음 낮잠 시간이 계산상 낮잠 길이를 기준으로 남을 수 있었다.

**원인 (Cause)**: 수동 시간 편집은 뒤 스케줄을 다시 계산했지만, 빠른 기록의 낮잠 종료 처리는 실제 기록만 저장하고 낮잠 스케줄 재계산 함수를 호출하지 않았다.

**해결 방법 (Solution)**: 빠른 기록으로 낮잠을 종료할 때도 실제 시작/종료 시각으로 낮잠 길이를 계산한 뒤, 같은 재계산 흐름을 적용해 뒤 낮잠 시간이 실제 종료 기준으로 갱신되도록 했다.

### [PROB-005] 다음 날에도 전날 루틴 수정값이 남음 (2026-07-15)

**문제 (Problem)**: 하루용 루틴 계산 도구인데, 전날 입력한 실제 기록과 시간 수정값이 다음 날에도 남을 수 있었다.

**원인 (Cause)**: 로컬 저장 데이터에 루틴 날짜를 구분하는 값이 없었다.

**해결 방법 (Solution)**: 저장 상태에 `routineDate`를 추가했다. 앱을 열었을 때 저장 날짜가 오늘과 다르면 기본 설정은 유지하고 스케줄과 실제 기록만 초기화하며, 앱을 켜둔 채 12시를 넘겨도 날짜 변경을 감지해 저장된 기준값으로 새 스케줄을 계산하도록 했다.

### [PROB-004] PWA 백그라운드 알림이 실기기에서 신뢰 불가 (2026-07-14)

**문제 (Problem)**: GitHub Pages에 배포된 토닥루틴도우미 PWA에서 앱 화면이 휴대폰에 떠 있지 않으면 일정 알림이 오지 않았다.

**원인 (Cause)**: 정적 PWA의 화면 타이머, Web Worker, Service Worker 타이머는 휴대폰 OS와 브라우저 백그라운드 정책에 의해 정지될 수 있다. 정해진 시각에 안정적으로 알림을 보내려면 네이티브 앱 로컬 알림 또는 서버 기반 Web Push가 필요하다.

**해결 방법 (Solution)**: 도우미 PWA에서 알림 권한 요청, 일정 알림 등록, 알림 테스트 UI와 관련 예약 코드를 제거했다. 알림은 토닥루틴 Flutter 본 앱 또는 서버 기반 Web Push 단계로 분리한다.

### [PROB-001] 화면을 띄워 둔 경우에만 일정 알림 수신 (2026-07-14)

**문제 (Problem)**: 앱 화면이 보이는 동안에는 일정 알림이 오지만, 홈 화면으로 내리거나 다른 앱을 보고 있으면 알림이 오지 않거나 크게 늦어졌다.

**원인 (Cause)**: 일정 예약이 브라우저 화면의 `window.setTimeout`에만 의존했다. 모바일 브라우저는 숨겨진 화면의 타이머를 제한하고, OS 절전 정책에 따라 앱 프로세스를 멈출 수 있다.

**해결 방법 (Solution)**: Web Worker와 Service Worker 기반 보조 예약을 시도했지만 실기기에서 신뢰성이 충분하지 않았다. 최종적으로 PROB-004에 따라 도우미 PWA의 알림 기능을 제거했다.

### [PROB-002] 알림 등록 후 일정 입력값이 바뀌는 문제 (2026-07-14)

**문제 (Problem)**: 첫기상, 첫수, 낮잠 길이 등을 나중에 수정할 수 있어 알림 등록 시점을 사용자가 판단하기 어려웠다.

**원인 (Cause)**: 수동 등록 버튼 문구 때문에 알림 시간이 최초 등록값에 고정되는 것처럼 보였다.

**해결 방법 (Solution)**: 이후 실기기 검증 결과에 따라 도우미 PWA의 알림 기능을 제거했다. 수동 등록 버튼과 관련 설정도 함께 삭제했다.

### [PROB-003] 알림 아이콘이 미리보기 배경을 포함함 (2026-07-14)

**문제 (Problem)**: 토닥루틴 알림 아이콘 시안이 어두운 미리보기 배경을 포함해 실제 앱 자산으로 바로 사용할 수 없었다.

**원인 (Cause)**: 디자인 확인용 이미지와 Android 알림용 단색 알파 자산이 분리되지 않았다.

**해결 방법 (Solution)**: 선택된 아기 얼굴, 머리 꼭대기 C자 컬, 궤도 실루엣은 유지하면서 흰색 단색 + 완전 투명 배경 PNG로 변환했다. 512px 마스터와 192px, 96px 파생 파일의 알파 채널과 RGB 값을 검증했다.
