# 아키텍처 - 토닥루틴도우미

## 개요

React와 TypeScript로 만든 모바일 우선 PWA입니다. 사용자의 기준 시간과 실제 기록으로 하루 루틴을 계산하고, 홈 화면 설치와 로컬 저장을 제공합니다.

## 핵심 구성

- `src/App.tsx`: 화면, 상태, 로컬 저장, 입력 이벤트
- `src/schedule.ts`: 수유/낮잠/이유식 일정 생성과 경고 계산
- `src/timeUtils.ts`: 시간 입력, 표시, 당일 기준 차이 계산
- `public/service-worker.js`: 설치형 PWA 갱신 보조
- `public/manifest.webmanifest`: 설치형 PWA 정보와 앱 아이콘

## 일정 계산 흐름

```text
사용자 입력/실제 기록
  -> 일정과 경고 재계산
  -> 낮잠/수유/이유식 카드 갱신
  -> 로컬 저장
  -> 날짜 변경 시 기본 설정은 유지하고 스케줄과 실제 기록만 새로 시작
```

실제 시간 기록을 입력하면 해당 카드와 뒤 일정이 즉시 갱신됩니다. 사용자가 입력한 값은 브라우저 로컬 저장소에 보관합니다.
저장 데이터에는 루틴 날짜를 함께 남기며, 앱을 열었을 때 저장 날짜가 오늘이 아니면 기본 설정은 유지하고 스케줄과 실제 기록만 초기화합니다. 앱을 켜둔 채 자정을 넘긴 경우에도 날짜 변경을 감지해 저장된 기준값으로 새 스케줄을 계산합니다.

## 알림 정책

토닥루틴도우미 PWA에서는 일정 알림 기능을 제공하지 않습니다. GitHub Pages 기반 정적 PWA는 휴대폰 백그라운드 알림을 안정적으로 보장하기 어렵기 때문입니다.

알림은 향후 토닥루틴 Flutter 본 앱의 로컬 알림 또는 서버 기반 Web Push 단계에서 다시 다룹니다.

## 자산 구조

- `public/brand/`: 실제 PWA 스플래시 및 별도 루틴 스플래시
- `public/icons/`: 실제 PWA 앱 아이콘
- `design-concepts/2026-07-13/`: 시안, 선택본, 전달용 원본
- `design-concepts/2026-07-13/routine-notification-production/`: 토닥루틴 투명 알림 PNG

## 설계 결정

### ADR-001: 도우미 PWA 알림 기능 제거 (2026-07-14)

- **결정**: 토닥루틴도우미에서는 알림 권한 요청, 일정 알림 등록, 알림 테스트 UI와 관련 예약 코드를 제거한다.
- **이유**: GitHub Pages 기반 정적 PWA는 휴대폰에서 앱 화면이 내려간 상태의 정시 알림을 안정적으로 보장하기 어렵다.
- **결과**: 도우미는 오늘 루틴 계산과 기록 흐름 검증에 집중한다. 알림은 토닥루틴 본 앱 또는 서버 푸시 단계로 분리한다.

### ADR-002: 날짜 변경 시 루틴 자동 초기화 (2026-07-15)

- **결정**: 저장 데이터에 루틴 날짜를 함께 저장하고, 오늘 날짜와 다르면 기본 설정은 유지한 채 스케줄과 실제 기록만 새로 시작한다.
- **이유**: 토닥루틴도우미는 하루 루틴 계산 도구라서 어제의 실제 기록과 시간 수정값이 다음 날 계산에 남으면 혼란스럽다.
- **결과**: 12시가 지나 앱을 다시 열거나 켜둔 상태에서 날짜가 바뀌면 첫기상, 첫수, 막수, 낮잠 길이 같은 기본 설정은 유지되고, 오늘 스케줄과 실제 기록만 새로 계산된다.

### ADR-003: 문서와 빌드 산출물 분리 (2026-07-14)

- **결정**: `docs/`는 문서 전용으로 유지하고 수동 Pages 빌드는 `dist-pages/`에 출력한다.
- **이유**: 빌드가 `docs/`의 CHANGELOG, ARCHITECTURE, SETUP을 삭제하는 충돌을 막기 위해서다.
- **결과**: GitHub Actions 배포는 기존처럼 `dist/`를 사용한다.
